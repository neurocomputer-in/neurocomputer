The Framework

The Neuro Framework.
A modular cognition substrate.

If a capability matters for intelligent behaviour, it should be representable as a neuro. If neuros are composable and reusable, the system becomes programmable intelligence. If programmable intelligence is scalable, we have a credible path toward agentic OS engineering.

The Unit

A neuro is the universal unit.

A neuro is a typed, composable runtime unit with identity, behaviour, and an optional language surface. The same shape covers thinking, planning, tool use, memory, model invocation, and behaviour policy.

conf · code · prompt

A neuro folder declares its identity in conf.json, its behaviour in code.py, and an optional prompt surface in prompt.txt. The factory hot-discovers folders at runtime — drop one in, the runtime sees it.

There is no atomic level the user must reach. Composites are first-class — a neuro can itself be a graph of neuros. The framework is fractally compositional.

neuros/extract_book/ ├── conf.json │ { │ "name": "extract_book", │ "kind": "skill.web", │ "effect": "tool", │ "inputs": { "url": "str" }, │ "output": "dict" │ } ├── code.py # async run(state) └── prompt.txt # optional
Ten Kinds

A taxonomy with intent.

Ten first-class neuro classes. Each has a base type, a folder convention, and a place in composition. Together they form the framework's vocabulary.

Model neuro
Provider/model abstraction. Hot-swappable backends (OpenAI, Anthropic, Ollama Gemma, local). core/model_neuro.py, core/llm_registry.py.
Prompt neuro
Prompt blocks and composers as reusable modules. Typed slots; composes categorically. core/prompt_neuro.py.
Instruction neuro
Rules, policy, tone as explicit behavioural constraints. core/instruction_neuro.py.
Context neuro
Context slicing and assembly as programmable operations. Typed I/O contracts. core/context_neuro.py.
Memory neuro
Store, recall, extraction, categorisation, consolidation. Layered memory logic. core/memory.py, core/memory_graph.py.
Skill neuro
Concrete capabilities — system ops, integrations, domain tools. The largest namespace.
Workflow neuro
Sequential, DAG, parallel, looped orchestration. core/flows/.
Agent neuro
Role-specialised orchestration units built from other neuros. core/agent_neuro.py.
Code neuro
Read, write, diff, patch, plan, scan operations for self-modification and coding workflows.
Library neuro
Registry, taxonomy, discoverability so neuros can be reused like language primitives.

This is why the framework is language-like. It has vocabulary (kinds and namespaces), grammar (composition operators), semantics (typed contracts), runtime (factory + brain + executor + event bus), and a standard library (built-in neuros and reusable domain libraries).

The Hierarchy

From neuro to agency.

Four layers. Each groups the next. The runtime is built so you can scale a thought from a single capability to an entire agency without changing the model.

L1 — Agency

Workspace + team

Color, emoji, set of agents, default project. core/agency.py, core/agency_configs.py. Shipped: Neuro HQ, Upwork, Web Agency.

L2 — Project

Unit of work

Lives inside an agency. Default project always present. core/project.py, core/project_configs.py, core/defaults.py.

L3 — Agent

Role orchestrator

Owns router, planner, replier, profile. core/agent.py, core/agent_manager.py. Shipped: neuro, nl_dev, opencode, openclaw, upwork.

L4 — Neuro

Atomic capability

The composable unit. Discovered by core/neuro_factory.py, all share core/base_neuro.py contract.

Orchestration

The Brain — Reason + Act.

Every message hits the brain first. In a single LLM call, the smart router classifies: direct reply, or skill execution? Simple things stay fast. Complex things get planned and executed.

Smart Router

The first stage. One LLM call asks: "is this conversational, or does it need a skill?" Direct reply → respond now. Skill needed → identify and pass to the planner.

Planner & DAG

For multi-step work, the planner builds a Directed Acyclic Graph of skill calls. Each node is one neuro invocation; outputs feed into later inputs. The executor walks the graph node by node.

User Input │ ▼ Router ─── "just chat?" ─── Direct Reply │ ▼ needs a skill Planner ─── builds DAG │ ▼ Executor ─── A → B → C │ ▼ Reply ─── final answer

Tool-loop neuro

Some tasks aren't a fixed DAG — they need iteration. The tool-loop neuro lets a neuro call tools in-reply, observe, and continue. Bounded, depth-guarded, observable.

Hot discovery

The factory rescans neurocomputer/neuros/ at runtime. Drop in a folder, it appears. Edit code, the next call uses the new version. No restart.

brain.handle(msg) │ ├── smart_router (intent) │ ├── planner (DAG) │ ├── executor (run nodes) │ └── reply (final) neuro_factory.discover() ├── walk neuros/ ├── load conf.json └── attach to registry
Multi-Agent

Rooms, talk, schedules.

Three primitives compose every multi-agent pattern: a typed talk primitive, persistent rooms with mediation, and time-aware triggers.

Rooms

Meeting Rooms

Multiple agents share a transcript. A mediator picks the next speaker round-robin. Visible in RoomPanel.tsx. Backed by rooms_db.py and four neuros: room_create, room_post, room_close, room_mediator. Endpoint: /api/rooms.

Talk

agent.talk(target, msg)

Direct typed messages between agents. Depth-guarded (MAX=4) to prevent infinite loops via TalkDepthExceeded. Two neuros: agent_talk, agent_list. The substrate beneath rooms; the seed of categorical mailbox protocols.

Schedules

Cron-style triggers

Persisted to schedules.db via APScheduler. Three neuros: schedule_run, schedule_cancel, schedule_list. "Set a timer", "every morning at 8 send me the summary" — scheduled from chat.

Profiles

Behaviour packs you can swap.

A profile is a behaviour pack — model choice, system prompt, defaults — applied at the agent level. Switch with one API call.

Profile Use Paired Agent
generalDefault conversational agent. All general-purpose skills available.neuro
code_devCode editing and project management focus.opencode / neuro
neuro_devAuthoring neuros inside Neurocomputer. Meta-programming.neuro
neurolang_devAuthoring NeuroLang flows. Wraps compile_source, propose_plan, decompile_summary.nl_dev
# Switch profile
POST /api/profile/switch  { "profile": "neurolang_dev" }
GET  /api/profile/active
GET  /api/profile/list
Authoring

Write a neuro in three files.

Add a folder under neurocomputer/neuros/<name>/. Declare its contract. Implement its behaviour. Test it. Hot-load.

neurocomputer/neuros/extract_book/
├── conf.json    # name, kind, inputs/outputs, model
├── code.py      # async def run(state, **kwargs) -> dict
└── prompt.txt   # optional, for LLM-driven neuros

Validate and run through the IDE backend or core/neuro_factory.py. Tests live in neurocomputer/tests/.

python3 neurocomputer/scripts/ide_server.py        # IDE backend, port 8000
cd neuro_web && NEXT_PUBLIC_IDE_URL=http://127.0.0.1:8000 npm run dev
# open http://localhost:3000/graph
Neurogramming

LEGO for coding.

The 3D Neuro IDE is not an accessory — it is the interaction model. Visualise neuros as graph-native programmable units. Inspect contracts, code, and prompts together. Compose visually and textually.

Graph-native

Built on @react-three/fiber. Nodes coloured by kind: skill, memory, model, prompt, agent. Drag to rearrange. Click to drill in.

Source-aware editing

Each neuro surfaces its conf.json, code.py, and prompt.txt together. Validation runs on save. Hot-load on success.

Safe save pipeline

Validation → snapshot → save → reload registry. If validation fails, the previous version stays live.

Reusable across projects

Build a neuro once; use it across agencies, projects, and agents. The library namespace makes discovery uniform.

Universality

Computationally complete.

The framework is computationally universal in direction. It supports state, branching, iterative execution, unbounded composition, and external I/O. General computation is expressible through neuro composition — not just prompt chaining.
