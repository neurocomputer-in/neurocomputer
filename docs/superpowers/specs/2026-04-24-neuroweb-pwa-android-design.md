# neuro_web PWA — Android Fullscreen App

**Date:** 2026-04-24
**Status:** Approved

## Goal

Make neuro_web installable on Android via Chrome as a Progressive Web App (PWA) that runs in fullscreen mode (equivalent to F11 on desktop) — no APK, no app store.

## Approach

Use `@ducanh2912/next-pwa` to layer PWA support onto the existing Next.js 14 App Router project. No structural changes to neuro_web. Electron desktop build is unaffected.

## Architecture

```
next.config.js        ← wrapped with withPWA() — generates sw.js at build time
public/manifest.json  ← PWA identity: name, icons, display: fullscreen
public/icons/         ← icon-192.png + icon-512.png
app/layout.tsx        ← <link rel="manifest"> + theme-color meta tag
```

The service worker is auto-generated by Workbox via the plugin. It is committed to `.gitignore` (build artifact).

## Caching Strategy

| Resource | Strategy | Reason |
|----------|----------|--------|
| JS/CSS/fonts (`/_next/static/`) | cache-first | Fast loads, versioned filenames |
| Static assets (`/icons/`, `/public/`) | cache-first | Immutable |
| API calls (`/api/*`, port 7000) | network-only | Always fresh, no stale AI data |

App shell loads fast from cache. All backend data (chat, terminal, agents) always hits live server. Offline = app shell visible, backend features show connection error.

## Fullscreen Behavior

`"display": "fullscreen"` in manifest hides address bar, browser chrome, and status bar on Android. User installs via Chrome menu → "Add to home screen" → app launches fullscreen from home screen icon.

## Out of Scope

- Offline support / background sync
- Push notifications
- neuro_mobile changes
- Icon branding (placeholder icons used initially)
