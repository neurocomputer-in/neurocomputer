# Phase 1.5 Bundle — Design Spec

**Status:** Drafted 2026-04-26. Pending user review, then writing-plans.

**Goal:** Ship the "registry is now real" milestone — three interlocking pieces that together make NeuroLang feel like a tool you'd actually use, not a one-shot LLM prompt.

The three pieces:

1. **Filesystem auto-discovery** — drop a `.py` in `~/.neurolang/neuros/` or `<project>/neuros/` and it's automatically registered on the next CLI invocation.
2. **Strict compiler validation** — `compile()` walks the generated source's AST and rejects (with helpful suggestions) any reference to a neuro that isn't actually registered.
3. **`neurolang plan` command** — propose a flow + cost estimate for a NL prompt before committing to compile, so the user sees what the AI is about to do.

Together these turn the current "type prompt → blind compile → hope" loop into "type prompt → see proposal → confirm → reliable compile."

---

## 1. Locked design decisions (from brainstorming)

| # | Decision | Choice |
|---|---|---|
| 1 | When does FS discovery scan? | Eager at CLI startup; explicit `discover()` for library users |
| 2 | Which directories get scanned? | `~/.neurolang/neuros/` + project-marker walk (look for `pyproject.toml` / `.neurolang` / `setup.py` / `.git`) + `<project>/neuros/` |
| 3 | How many LLM calls per `plan`? | One. LLM proposes neuros + composition + missing-capability list; cost rolled up locally from `Budget` metadata |
| 4 | `plan` UX flow? | TTY-aware hybrid: interactive prompt when on a TTY; `--yes` / `--no-confirm` / `--dry-run` flags for scripts |
| 5 | Strict validation: what counts as a reference? | Walk `flow = <expr>` rhs; for each `Name` / `Attribute`, check candidate name against registry first; if not registered but the head is locally defined (imported, assigned, def'd), accept and defer to runtime; otherwise flag with suggestions |
| 6 | Suggestion algorithm? | Hybrid: `difflib.get_close_matches` (Levenshtein) + substring search across name + description, deduped, top 3 |
| 7 | `plan` output format? | JSON canonical (pretty-printed `indent=2`); human X-ray UI is a downstream concern (Neurocomputer IDE / future viewer) |

---

## 2. Architecture

Three new modules + two extended modules:

```
neurolang/
├── discover.py     [NEW]    — filesystem auto-discovery
├── propose.py      [NEW]    — NL → ProposedPlan (single-LLM-call core)
├── suggest.py      [NEW]    — hybrid suggestions (small util)
├── compile.py      [EXTEND] — strict_refs option in validate_source(); walks flow=… rhs
├── cli.py          [EXTEND] — auto-runs discover() at startup; new `plan` subcommand
└── tests/
    ├── test_discover.py [NEW]
    ├── test_propose.py  [NEW]
    ├── test_suggest.py  [NEW]
    ├── test_compile.py  [EXTEND] — strict validation cases
    └── test_cli.py      [NEW]    — small CLI integration
```

Module boundaries:

- `discover.py` knows about filesystem + project-marker walk. Doesn't know about LLMs or compiling.
- `propose.py` knows about catalogs + LLM proposals + cost rollup. Doesn't know about CLI or filesystem.
- `suggest.py` is a pure utility — input `(name, registry) → list[str]`. Reused by validation and propose.
- `compile.py` extends its existing `validate_source()` — strict mode is a flag, default still permissive (don't break Phase 1.1).
- `cli.py` orchestrates: on every CLI entrypoint, calls `discover()` first, then dispatches to subcommand handlers.

Each module is independently testable with pure-Python mocks (no real LLM, no real disk for unit tests).

## 3. Component contracts

### 3.1 `discover.py`

```python
from dataclasses import dataclass, field
from pathlib import Path

@dataclass
class DiscoveryReport:
    user_dir_neuros:    list[Path]   # files imported from ~/.neurolang/neuros/
    project_dir_neuros: list[Path]   # files imported from <project>/neuros/
    extra_neuros:       list[Path]   # files from extra_paths
    project_root:       Path | None  # detected project root, if any
    errors:             list[tuple[Path, str]] = field(default_factory=list)

def discover(extra_paths: list[Path] | None = None) -> DiscoveryReport:
    """Scan ~/.neurolang/neuros/ + project-detected ./neuros/ + extra_paths.
    Imports each .py file (side-effect populates default_registry via @neuro).
    Idempotent: tracks already-imported absolute paths in a module-level set."""

def find_project_root(start: Path | None = None) -> Path | None:
    """Walk up from `start` (default cwd) looking for `.neurolang`, `pyproject.toml`,
    `setup.py`, or `.git`. Returns the first match, or None if walk reaches /."""
```

**Why this shape:**

- `DiscoveryReport` makes failures visible (one bad neuro file doesn't kill discovery; CLI surfaces the errors only on `--show-discovery` flag).
- `extra_paths` is the seam for tests (point at fixture dir) and for a future `NEUROLANG_PATH` env var.
- Idempotent guard prevents double-registration if `discover()` is called multiple times in the same process.
- `find_project_root` is exposed because `propose` may want to display "operating in project: /home/x/y" in JSON output.

**Marker priority for `find_project_root`** (first match wins):

1. `.neurolang` (sentinel file or dir — explicit opt-in)
2. `pyproject.toml`
3. `setup.py`
4. `.git`

`.neurolang` first because it's the explicit "this IS a NeuroLang project" signal; the others are conventional Python markers used as fallbacks.

### 3.2 `propose.py`

```python
from dataclasses import dataclass
from pathlib import Path

@dataclass(frozen=True)
class NeuroChoice:
    name: str
    effects: list[str]
    cost_estimate_usd: float    # from Budget metadata, 0.0 if undeclared
    latency_estimate_ms: int    # from Budget metadata, 0 if undeclared

@dataclass(frozen=True)
class MissingCapability:
    intent: str             # one short phrase from the LLM, e.g. "send an email"
    suggestions: list[str]  # closest neuro names from suggest.suggest_alternatives()

@dataclass(frozen=True)
class ProposedPlan:
    prompt: str                       # the user's NL input
    composition_source: str           # the proposed `flow = ...` Python source (one expr)
    neuros: list[NeuroChoice]         # the catalog neuros it would use
    missing: list[MissingCapability]  # what couldn't be satisfied
    cost_estimate_usd: float          # sum of neuros' cost_estimate_usd
    latency_estimate_ms: int          # sum of neuros' latency_estimate_ms (sequential assumption)
    project_root: Path | None         # for context

def propose(
    prompt: str,
    *,
    model: str = "openai",
    llm_fn=None,
    registry=None,
) -> ProposedPlan:
    """One LLM call. Catalog → LLM returns:
      {neuros: [name, ...], composition: 'flow = a | b | c',
       missing: [{intent: '...'}]}
    We then locally:
      - look up each neuros[i].name in the registry
      - pull cost_estimate_usd + latency_estimate_ms from each neuro's Budget
      - call suggest.suggest_alternatives() for each missing[i].intent
      - sum the cost/latency
    """

class ProposeError(Exception):
    """Raised on invalid LLM JSON or unparseable proposal."""
```

**Why this shape:**

- `ProposedPlan` is JSON-serializable via `dataclasses.asdict` → trivial CLI output.
- LLM is asked for *structure only* (neuros + composition + missing). Numeric estimates are computed deterministically in Python from registry metadata. LLM for creative work, Python for math.
- `composition_source` is a Python source snippet — the same string `compile()` would produce. `plan` then `compile` is just `compile(plan.prompt)`; cache may or may not hit (the two LLM calls have different system prompts).

**Latency rollup assumption:** sequential composition (`|`). The proposal rarely uses `&` / `+` in Phase 1.5; when it does, latency-rollup will be over-estimated (sum instead of max). Acceptable for v1; refine in a later phase if needed.

### 3.3 `suggest.py`

```python
def suggest_alternatives(
    unknown_name: str,
    registry,
    *,
    top_k: int = 3,
    levenshtein_cutoff: float = 0.6,
) -> list[str]:
    """Hybrid suggestion algorithm.
    1. difflib.get_close_matches(unknown_name, all_names, n=top_k, cutoff=levenshtein_cutoff)
    2. Substring search: any neuro whose name OR description contains a token of unknown_name
       (split on '.' and '_')
    3. Combine, dedup, return top_k preserving order (Levenshtein hits first)."""
```

Pure function, no side effects, easy to unit test.

### 3.4 `compile.py` extension

```python
from dataclasses import dataclass, field

@dataclass(frozen=True)
class ValidationFindings:
    imports: list[str]
    has_neurolang_import: bool
    declares_flow: bool
    unknown_refs: list[str] = field(default_factory=list)            # only if strict_refs=True
    suggestions: dict[str, list[str]] = field(default_factory=dict)  # {unknown: [alt, alt, alt]}

def validate_source(
    source: str,
    *,
    registry=None,
    strict_refs: bool = False,
) -> ValidationFindings:
    """Existing checks (parse, neurolang import, declares flow) plus, if strict_refs=True:
       - pre-pass: collect names defined locally in the source above the
         `flow = ...` line (any Assign, FunctionDef, ClassDef, Import, ImportFrom)
       - locate the `flow = <expr>` assignment
       - walk <expr> collecting Name and Attribute nodes; resolve each to a
         candidate name string (bare Name → 'X'; Attribute chain → 'X.Y.Z')
       - for each candidate:
           * if candidate is in registry (full match)            → OK
           * else if the head (bare Name or Attribute base) is in
             the locally-defined set                              → OK (defer to runtime)
           * else                                                 → unknown_ref
       - call suggest.suggest_alternatives() for each unknown_ref
    """

# compile.compile() hardcodes strict_refs=True internally (the new behavior).
# Direct callers of validate_source() keep strict_refs=False default for backward compat
# (so Phase 1.1 tests calling validate_source() directly don't break).
```

**Why a flag (not always-on):**

- Phase 1.1 tests use mock LLMs returning sources that may reference things outside the test registry. Forcing strict-refs would break those.
- `compile()` itself opts in to strict mode; users calling `validate_source()` directly keep the lax default.

### 3.5 `cli.py` extension

```python
def main(argv=None) -> int:
    parser = ...
    # Add `plan` subparser BEFORE parsing args
    pp = sub.add_parser("plan", help="Propose a flow for a natural-language prompt")
    pp.add_argument("prompt")
    pp.add_argument("--model", default="openai", choices=["openai", "anthropic"])
    pp.add_argument("--yes", "--no-confirm", action="store_true",
                    help="Skip the TTY confirmation; proceed directly to compile")
    pp.add_argument("--dry-run", action="store_true",
                    help="Print proposal as JSON; do not compile")
    pp.add_argument("--execute", action="store_true",
                    help="After compile, also run the flow")
    pp.add_argument("-o", "--output-file",
                    help="Write compiled source to this file")
    pp.add_argument("--show-discovery", action="store_true",
                    help="Print the DiscoveryReport before dispatching")
    pp.set_defaults(func=_cmd_plan)

    args = parser.parse_args(argv)

    # Run discovery before dispatch (idempotent on repeated calls)
    from . import discover
    report = discover.discover()
    if getattr(args, "show_discovery", False):
        print(json.dumps({"discovery": dataclasses.asdict(report)}, indent=2, default=str))

    return args.func(args)
```

`_cmd_plan` flow:

1. Call `propose.propose(args.prompt, model=args.model)` → `ProposedPlan`.
2. Print `json.dumps(dataclasses.asdict(plan), indent=2, default=str)` to stdout.
3. Branch on flags / TTY:
   - `--dry-run`: exit 0.
   - `--yes` (or stdin not a TTY): proceed to compile.
   - TTY + no `--yes`: prompt `Compile this plan? [y/N/e] `.
     - `y`: proceed to compile.
     - `n` (default): exit 0.
     - `e`: exit with hint `"Re-run with edited prompt: neurolang plan \"<edited>\""` (no editor integration in v1).
4. Compile path: `compile.compile(args.prompt, model=args.model, output="both")` → (flow, source).
   - Write source to `--output-file` if given, else stdout.
   - If `--execute`: `flow.run()`; print result.

## 4. Data flow end-to-end

```
$ neurolang plan "fetch unread emails and summarize them"
            │
            ▼
┌────────────────────────────────────────────────────────────────────────┐
│ cli.main()                                                             │
│  1. discover.discover() — find_project_root() + import every .py       │
│     in ~/.neurolang/neuros/ + <project>/neuros/                        │
│     - @neuro decorator side-effects populate default_registry          │
│     - errors collected into DiscoveryReport (silent unless --show-...) │
│  2. dispatch _cmd_plan(args)                                           │
└──────────────────────────────────┬─────────────────────────────────────┘
                                   ▼
┌────────────────────────────────────────────────────────────────────────┐
│ propose.propose(prompt, model)                                         │
│  1. _render_catalog(default_registry) → markdown of all neuros         │
│  2. system = _PLAN_SYSTEM_PROMPT + catalog + few_shot                  │
│  3. LLM call (json output expected)                                    │
│     LLM returns: {neuros, composition, missing}                        │
│  4. parse JSON; for each neuros[i].name: lookup + pull Budget          │
│  5. for each missing[i]: suggest.suggest_alternatives()                │
│  6. return ProposedPlan dataclass                                      │
└──────────────────────────────────┬─────────────────────────────────────┘
                                   ▼
┌────────────────────────────────────────────────────────────────────────┐
│ _cmd_plan emits JSON (pretty-printed, indent=2):                       │
│ {                                                                      │
│   "prompt": "...",                                                     │
│   "composition_source": "flow = web.search | reason.summarize",        │
│   "neuros": [                                                          │
│     {"name": "web.search", "effects": ["tool"],                        │
│      "cost_estimate_usd": 0.001, "latency_estimate_ms": 500},          │
│     {"name": "reason.summarize", "effects": ["llm"],                   │
│      "cost_estimate_usd": 0.005, "latency_estimate_ms": 1500}          │
│   ],                                                                   │
│   "missing": [                                                         │
│     {"intent": "fetch unread emails",                                  │
│      "suggestions": ["web.search", "web.scrape"]}                      │
│   ],                                                                   │
│   "cost_estimate_usd": 0.006,                                          │
│   "latency_estimate_ms": 2000,                                         │
│   "project_root": "/home/ubuntu/neurolang"                             │
│ }                                                                      │
└──────────────────────────────────┬─────────────────────────────────────┘
                                   ▼
┌────────────────────────────────────────────────────────────────────────┐
│ Branch on flags / TTY:                                                 │
│  --dry-run         → exit 0                                            │
│  --yes / piped     → call compile.compile(prompt) [+ --execute]        │
│  TTY + no --yes    → prompt "Compile this plan? [y/N/e]"               │
│                       y → compile [+ --execute]                        │
│                       N → exit 0                                       │
│                       e → exit with hint                               │
└──────────────────────────────────┬─────────────────────────────────────┘
                                   ▼
┌────────────────────────────────────────────────────────────────────────┐
│ compile.compile(prompt) [strict_refs=True]                             │
│  1. cache lookup                                                       │
│  2. if miss: LLM call → source; cache forward                          │
│  3. validate_source(source, strict_refs=True):                         │
│       - parse                                                          │
│       - find `flow = <expr>`; walk Names + Attributes                  │
│       - classify: neurolang-imported, local @neuro, unknown            │
│       - for each unknown: suggest_alternatives()                       │
│       - if unknown_refs non-empty → CompileError with suggestions      │
│  4. exec source → return Flow                                          │
└────────────────────────────────────────────────────────────────────────┘
```

**Cache linkage between `plan` and `compile`:** `propose.py` uses a `propose:` cache key prefix; `compile.py` uses its existing keying. They are separate cache namespaces. Picking `y` after `plan` triggers a fresh `compile()` call. Tradeoff: the proposal and the compiled source could theoretically diverge (different system prompts → different LLM outputs). Acceptable because each prompt stays focused on one concern (proposal needs structure JSON; compile needs Python source).

## 5. Error handling

### 5.1 Discovery (non-fatal)

A bad neuro file should never kill the CLI.

```python
# In discover.discover():
for path in candidate_paths:
    try:
        spec = importlib.util.spec_from_file_location(...)
        mod  = importlib.util.module_from_spec(spec)
        spec.loader.exec_module(mod)
    except Exception as e:
        report.errors.append((path, f"{type(e).__name__}: {e}"))
        continue
```

CLI surfaces them only on `--show-discovery` (default: silent). Errors stay in `DiscoveryReport` for programmatic access.

### 5.2 Propose

- **LLM returns invalid JSON** → `propose()` raises `ProposeError`. CLI catches, prints structured error JSON:
  ```json
  {"error": "propose_failed", "stage": "json_parse", "message": "...", "raw_llm_output": "..."}
  ```
  Exit code: `2`.
- **LLM returns valid JSON but references nonexistent neuros in `composition`** → those refs are flagged as `missing` capabilities (not errors). The proposal still emits.
- **Empty registry** → `propose()` returns a `ProposedPlan` with `missing` populated for every intent and `neuros: []`.

### 5.3 Strict validation

`compile(strict_refs=True)` finds unknown refs → raises `CompileError`:

```
CompileError: Generated source references unknown neuros.

Unknown:  email.fetch
  Did you mean? web.search, web.scrape

Unknown:  email.send
  No close matches in registry.

Source:
  flow = email.fetch | reason.summarize | email.send
```

Exit code: `1`. Source is shown so the user can hand-edit.

### 5.4 TTY-prompt

- Stdin closed mid-prompt → treat as `n` (cancel). EOFError swallowed.
- `--yes` + `--dry-run` together → CLI rejects with usage error before any LLM call.

### 5.5 Discovery edge cases

- `cwd` not under any project root → `project_root: None`, only `~/.neurolang/neuros/` scanned. No error.
- Project root found but no `<project>/neuros/` dir → no error, just `project_dir_neuros: []`.
- Same neuro name registered twice (one from user dir, one from project dir) → project wins (registry's `add()` is last-write). Surfaced via `DiscoveryReport.errors` as a warning.

## 7. Out of scope (explicit)

- Embedding/semantic search for suggestions — Phase 2 (per ROADMAP.md).
- Watch-mode FS discovery (re-scan on file change) — defer.
- `NEUROLANG_PATH` env var — defer; `extra_paths=` arg already present as the seam.
- Editor integration on `e` choice in TTY prompt — defer; v1 just hints at re-running.
- Structured-output enforcement (Outlines / Instructor) — Phase 2.
- `plan --json` flag toggle — `plan` is JSON-only by design (Q7); no flag needed.
- Persistent file-cache for `propose` — propose has its own file cache via the same `CompilerCache` infrastructure (`propose:` key prefix), but no separate management surface in v1.
